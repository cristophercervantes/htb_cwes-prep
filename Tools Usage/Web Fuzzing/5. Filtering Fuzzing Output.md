# Filtering Fuzzing Output 

> Fuzzing tools will happily dump thousands of results at you. Most of it is noise. Filtering is what turns an overwhelming wall of 404s into something actually useful.

Running ffuf without any filters defaults to matching these status codes: `200-299, 301, 302, 307, 401, 403, 405, 500`. That's actually pretty reasonable — it cuts out 404s automatically.

But if you run with `-mc all` (match everything), you get absolutely buried:

```
[Status: 404, Size: 36, Words: 4, Lines: 3] → .cache
[Status: 404, Size: 43, Words: 4, Lines: 3] → .bash_history
[Status: 404, Size: 34, Words: 4, Lines: 3] → .cvs
... (thousands more)
```

Every single wordlist entry gets reported. Finding anything meaningful in that is basically impossible. Filtering = sanity.

## Gobuster

Pretty straightforward filtering, but with a caveat: `-s` and `-b` are **only available in `dir` mode**.

|Flag|What it does|
|---|---|
|`-s`|Include only these status codes (comma-separated)|
|`-b`|Exclude these status codes|
|`--exclude-length`|Exclude responses of specific sizes (supports ranges)|

```bash
# Only show 200 and 301, skip empty responses
gobuster dir -u http://example.com/ -w wordlist.txt -s 200,301 --exclude-length 0
```

## ffuf

ffuf has the most flexible filtering of the bunch. Filters work on status codes, size, word count, line count, and even response time.

| Flag  | Include or Exclude | Filters on           |
| ----- | ------------------ | -------------------- |
| `-mc` | nclude (match)     | Status codes         |
| `-fc` | Exclude (filter)   | Status codes         |
| `-ms` | Include            | Response size        |
| `-fs` | Exclude            | Response size        |
| `-mw` | Include            | Word count           |
| `-fw` | Exclude            | Word count           |
| `-ml` | Include            | Line count           |
| `-fl` | Exclude            | Line count           |
| `-mt` |  Include           | Response time (TTFB) |

A few practical examples:

```bash
# Only 200 responses, skip 427-word responses, must be >500 bytes
ffuf -u http://example.com/FUZZ -w wordlist.txt -mc 200 -fw 427 -ms >500

# Drown out the noise — hide 404, 401, 302
ffuf -u http://example.com/FUZZ -w wordlist.txt -fc 404,401,302

# Looking for a .bak file between 10KB and 100KB
ffuf -u http://example.com/FUZZ.bak -w wordlist.txt -fs 0-10239 -ms 10240-102400

# Slow endpoints — could indicate interesting processing
ffuf -u http://example.com/FUZZ -w wordlist.txt -mt >500
```

> The response time filter (`-mt`) is one I'd forget about, but it's actually useful — slow responses on specific inputs can hint at SQL injection, heavy processing, or something worth poking at.

## wenum

wenum's naming convention is pretty consistent once you see the pattern: `--hc` = hide code, `--sc` = show code, and so on.

| Flag   | Include or Exclude | Filters on          |
| ------ | ------------------ | ------------------- |
| `--sc` | Include            | Status codes        |
| `--hc` | Exclude            | Status codes        |
| `--sl` | Include            | Line count          |
| `--hl` | Exclude            | Line count          |
| `--sw` | Include            | Word count          |
| `--hw` | Exclude            | Word count          |
| `--ss` | Include            | Response size       |
| `--hs` | Exclude            | Response size       |
| `--sr` | Include            | Regex match in body |
| `--hr` |  Exclude           | Regex match in body |

The regex flags are particularly useful — `--sr "admin"` will only show responses that contain the word "admin" in the body. `--hr "Internal Server Error"` hides those annoying error pages.

```bash
# Only show 200s and redirects
wenum -w wordlist.txt --sc 200,301,302 -u https://example.com/FUZZ

# Hide common junk
wenum -w wordlist.txt --hc 404,400,500 -u https://example.com/FUZZ

# Looking for something specific in the response body
wenum -w wordlist.txt --sr "admin\|password" -u https://example.com/FUZZ
```

## feroxbuster

feroxbuster splits it clearly: `-s` is an allowlist (only show these), `-C` is a denylist (hide these). It also has a handy `--filter-similar-to` flag for removing near-duplicate pages.

| Flag                  | Include or Exclude | Filters on                              |
| --------------------- | ------------------ | --------------------------------------- |
| `-s`                  | Include            | Status codes (allowlist)                |
| `-C`                  | Exclude            | Status codes (denylist)                 |
| `-S`                  | Exclude            | Response size                           |
| `-W`                  | Exclude            | Word count                              |
| `-N`                  | Exclude            | Line count                              |
| `-X`                  | Exclude            | Regex match in body/headers             |
| `--filter-similar-to` | Exclude            | Similar pages to a reference            |
| `--dont-scan`         |  Skip entirely     | URLs/patterns (won't even request them) |

```bash
# 200 only, skip anything over 10KB, hide pages with "error" in them
feroxbuster --url http://example.com -w wordlist.txt -s 200 -S 10240 -X "error"
```
