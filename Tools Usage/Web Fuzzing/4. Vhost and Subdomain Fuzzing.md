## Virtual Host & Subdomain Fuzzing

**Virtual Hosts** — Multiple websites sharing one server/IP. The server decides which site to serve based on the **Host header** in the HTTP request.

**Subdomains** — Extensions of a main domain (e.g. `blog.example.com`) resolved through **DNS records** to specific IP addresses.

| Feature       | Virtual Hosts                             | Subdomains                            |
| ------------- | ----------------------------------------- | ------------------------------------- |
| Identified by | Host header                               | DNS records                           |
| Same server?  | Yes                                       | Not always                            |
| Security risk | Misconfigured vhosts expose internal apps | Forgotten DNS records can be hijacked |

### Gobuster — VHost Fuzzing

**Setup:**

```bash
echo "IP inlanefreight.htb" | sudo tee -a /etc/hosts
```

**Command:**

```bash
gobuster vhost -u http://inlanefreight.htb:81 \
               -w /usr/share/seclists/Discovery/Web-Content/common.txt \
               --append-domain
```

**Flags:**

- `vhost` → vhost discovery mode
- `-u` → target URL
- `-w` → wordlist
- `--append-domain` → builds full names like `admin.inlanefreight.htb`

**What to look for:**

```
Found: admin.inlanefreight.htb:81   Status: 200  Valid
Found: .git/logs/.inlanefreight.htb Status: 400  Ignore
```

Only care about **Status 200**.

### Gobuster — Subdomain Fuzzing

**Command:**

```bash
gobuster dns -d inlanefreight.com \
             -w /usr/share/seclists/Discovery/DNS/subdomains-top1million-5000.txt
```

**Flags:**

- `dns` → DNS enumeration mode
- `-d` → target domain (use `--domain` in newer versions)
- `-w` → wordlist

**Output:**

```
Found: www.inlanefreight.com
Found: blog.inlanefreight.com
```
