## **Web Fuzzing**

Web fuzzing is basically throwing a bunch of random/unexpected inputs at a web app to see what breaks. It's a core skill in web security.

**Fuzzing vs Brute-forcing** People use these terms interchangeably and that's fine. The difference is:

- **Fuzzing** = throwing anything and everything at the app (random data, weird characters, garbage inputs) to see how it reacts
- **Brute-forcing** = systematically trying known values like passwords or IDs until one works

Think of it like a locked door — fuzzing is throwing random objects at it, brute-forcing is trying every key one by one.

**Reason for Fuzzing:** Manual testing misses a lot. Fuzzing automates the boring part — sending thousands of inputs — so you can focus on what actually breaks. It catches hidden bugs, weak input validation, and things like SQLi or XSS entry points before attackers find them. You can even plug it into your CI/CD pipeline so it runs automatically.

**Important Terms**

- **Wordlist** — your list of inputs to try (e.g. admin, login, config)
- **Payload** — the actual data you're sending
- **Response analysis** — reading the app's reaction (500 error? interesting.)
- **Fuzzer** — the tool doing all this (ffuf, wfuzz, Burp Suite)
- **False positive** — fuzzer thinks it found something, it didn't
- **False negative** — real vulnerability the fuzzer missed
- **Scope** — which part of the app you're targeting

**Web Fuzzing Tools — Setup 

**First — install Go, Python, and pipx**

pipx is just a cleaner way to install Python tools — it keeps each tool in its own isolated environment so nothing conflicts with each other.

On any Debian-based system (Ubuntu, Kali, etc.):

```bash
sudo apt update
sudo apt install -y golang
sudo apt install -y python3 python3-pip
sudo apt install pipx
pipx ensurepath
sudo pipx ensurepath --global
```

Verify everything installed correctly:

```bash
go version
python3 --version
```

**The four tools**

**ffuf** (Fuzz Faster U Fool) — the most popular one. Written in Go, very fast. Great for directory discovery, parameter fuzzing, and brute-forcing. You'll use this the most.

```bash
go install github.com/ffuf/ffuf/v2@latest
```

**Gobuster** — similar to ffuf but simpler. Good for beginners. Also handles DNS subdomain enumeration which ffuf doesn't do as cleanly.

```bash
go install github.com/OJ/gobuster/v3@latest
```

**FeroxBuster** — written in Rust so it's very fast. Unique feature is recursive scanning — it automatically goes deeper into directories it finds.

```bash
curl -sL https://raw.githubusercontent.com/epi052/feroxbuster/main/install-nix.sh | sudo bash -s $HOME/.local/bin
```

**wenum** (fork of wfuzz) — best for parameter fuzzing specifically. If you're on Kali, wfuzz might already be installed. wenum is a drop-in replacement with the same syntax.

```bash
pipx install git+https://github.com/WebFuzzForge/wenum
pipx runpip wenum install setuptools
```
