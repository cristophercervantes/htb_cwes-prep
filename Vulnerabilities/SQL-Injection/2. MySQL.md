# Intro to MySQL

SQL syntax varies slightly between databases but they all follow the ISO standard. MySQL/MariaDB is what we're using here.

## Connecting to MySQL

```bash
mysql -u root -p
```

Use `-p` without the password right after it — you'll be prompted to enter it. If you put the password directly in the command it ends up in your bash history.

For a remote host:

```bash
mysql -u root -h docker.labenv.eu -P 3306 -p
```

Capital `-P` for port, lowercase `-p` for password. Easy to mix up.

## Basic database commands

```sql
CREATE DATABASE users;
SHOW DATABASES;
USE users;
```

SQL keywords aren't case-sensitive (`use` and `USE` both work) but database names are — `USE USERS` won't find `users`.

## Creating a table

```sql
CREATE TABLE logins (
    id INT NOT NULL AUTO_INCREMENT,
    username VARCHAR(100) UNIQUE NOT NULL,
    password VARCHAR(100) NOT NULL,
    date_of_joining DATETIME DEFAULT NOW(),
    PRIMARY KEY (id)
);
```

Breaking down the column properties:

- `NOT NULL` — field can't be empty
- `AUTO_INCREMENT` — id goes up by 1 automatically on each new row
- `UNIQUE` — no duplicates allowed (useful for usernames)
- `DEFAULT NOW()` — fills in current timestamp if nothing is provided
- `PRIMARY KEY` — uniquely identifies each row

Check what's in a table with:

```sql
SHOW TABLES;
DESCRIBE logins;
```

# SQL Statements

## INSERT

```sql
-- all columns
INSERT INTO logins VALUES(1, 'admin', 'p@ssw0rd', '2020-07-02');

-- specific columns only (skips id and date_of_joining which have defaults)
INSERT INTO logins(username, password) VALUES('administrator', 'adm1n_p@ss');

-- multiple rows at once
INSERT INTO logins(username, password) VALUES ('john', 'john123!'), ('tom', 'tom123!');
```

Skipping a `NOT NULL` column with no default will throw an error.

## SELECT

```sql
SELECT * FROM logins;                        -- everything
SELECT username, password FROM logins;       -- specific columns only
```

## DROP

```sql
DROP TABLE logins;
```

Gone instantly, no confirmation prompt. Be careful.

## ALTER

For changing the table structure itself — adding, renaming, retyping, or dropping columns.

```sql
ALTER TABLE logins ADD newColumn INT;
ALTER TABLE logins RENAME COLUMN newColumn TO newerColumn;
ALTER TABLE logins MODIFY newerColumn DATE;
ALTER TABLE logins DROP newerColumn;
```

## UPDATE

```sql
UPDATE logins SET password = 'change_password' WHERE id > 1;
```

Always use `WHERE` — without it you update every single row in the table.

# Query Results

## ORDER BY

```sql
SELECT * FROM logins ORDER BY password;          -- ascending by default
SELECT * FROM logins ORDER BY password DESC;     -- descending
SELECT * FROM logins ORDER BY password DESC, id ASC;  -- sort by two columns
```

## LIMIT

```sql
SELECT * FROM logins LIMIT 2;        -- first 2 rows
SELECT * FROM logins LIMIT 1, 2;     -- skip 1, then return 2 rows
```

`LIMIT offset, count` — offset starts at 0, so `LIMIT 1, 2` skips the first row and returns the next two.

## WHERE

```sql
SELECT * FROM logins WHERE id > 1;
SELECT * FROM logins WHERE username = 'admin';
```

Strings need quotes, numbers don't.

## LIKE

```sql
SELECT * FROM logins WHERE username LIKE 'admin%';  -- starts with "admin"
SELECT * FROM logins WHERE username LIKE '___';      -- exactly 3 characters
```

- `%` — matches zero or more characters
- `_` — matches exactly one character

# SQL Operators

## AND / OR / NOT

```sql
SELECT * FROM logins WHERE username != 'john';
SELECT * FROM logins WHERE username != 'john' AND id > 1;
```

- `AND` — both conditions must be true
- `OR` — at least one must be true
- `NOT` / `!=` — flips or negates

Symbol equivalents: `&&`, `||`, `!`

In MySQL, `1` = true, `0` = false.

## Operator precedence (high to low)

```
*, /, %
+, -
=, >, <, >=, <=, !=, LIKE
NOT (!)
AND (&&)
OR (||)
```

So in `WHERE username != 'tom' AND id > 3 - 2`, the subtraction runs first → becomes `id > 1`, then the comparisons, then AND.

Useful to know when reading or crafting injection payloads — OR has lower precedence than AND, which affects how conditions chain together.
