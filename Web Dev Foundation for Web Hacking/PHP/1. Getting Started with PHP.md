# What is PHP 

PHP is a **server-side scripting language** built specifically for web development. It also works as a general-purpose language — meaning it's not locked into just one thing.

The name went through a change:

- Originally: **Personal Home Page**
- Now: **Hypertext Preprocessor** _(yes, it's recursive — PHP is in its own acronym)_

Created by **Rasmus Lerdorf** in **1994**, now maintained by the PHP Development Team.

## Server-side 

When you visit a website, here's what quietly happens:

1. Your browser sends an HTTP request to a web server
2. The server runs the PHP code
3. PHP generates an HTML document
4. That HTML gets sent back to your browser

Your browser never sees the PHP code — it only sees the final HTML output. PHP lives and runs on the **server**, not in your browser.

## General-purpose vs domain-specific

Some languages are built for one job only — like **SQL**, which is purely for querying databases. You can't build a web app with SQL alone.

PHP is different. It's a **general-purpose language**, meaning you can build all kinds of things with it — websites, command-line tools, PDF generators, image processors, etc.

## PHP runs everywhere 

- Works on **Linux, Windows, macOS**
- Compatible with **Apache, Nginx, OpenBSD**
- Runs on cloud platforms like **AWS and Azure**
- Can generate **PDFs, GIFs, JPEGs, PNGs**
- Talks to databases like **MySQL, PostgreSQL, MongoDB, Oracle, MS SQL**

## Two main things PHP is used for

1. **Server-side scripting** — building dynamic websites and web apps
2. **Command-line scripting** — running tasks like sending emails or generating files (similar to Python or Perl)

## How PHP Works (the flow)

```
Browser  →  HTTP Request (index.php)
              ↓
         Web Server
              ↓
         PHP Preprocessor processes the .php file
              ↓
         Generates HTML
              ↓
Browser  ←  HTML Response
```

The browser only ever receives plain HTML. All the logic stays on the server. That's a big deal from a security perspective — and why PHP misconfigurations are so interesting to hackers.

# 02 — PHP Setup 
## Three ways to get PHP running

|Method|Best for|Effort|
|---|---|---|
|XAMPP|Windows/macOS, absolute beginners|Very low|
|Manual Linux|Learning how the pieces connect|Medium|
|Docker|Clean, portable, repeatable environments|Low (once you know Docker)|

## Method 1: XAMPP 

XAMPP bundles **Apache + PHP + MySQL + phpMyAdmin** into one installer. No terminal needed.

**Steps:**

1. Download from [apachefriends.org](https://www.apachefriends.org/)
2. Run installer, keep all components selected
3. Open XAMPP Control Panel → Start **Apache** and **MySQL**
4. Visit `http://localhost` — you should see the XAMPP dashboard

**Quick PHP test:**

```
Create: htdocs/index.php
```

```php
<?php
phpinfo();
?>
```

Visit `http://localhost` — if you see a purple table of PHP settings, you're good.

**Your code goes in:** `htdocs/` folder  
**Database manager:** `http://localhost/phpmyadmin`

If Apache won't start — something is already using **port 80** (Skype and IIS are common culprits). Change Apache's port in XAMPP settings, or close the conflicting app.

## Method 2: Manual Setup on Linux 

Installing each piece manually means you understand how it all connects. Great for learning.

```bash
# 1. Update system
sudo apt update && sudo apt upgrade

# 2. Install Apache
sudo apt install apache2
sudo systemctl status apache2   # check it's running

# 3. Install PHP
sudo apt install php libapache2-mod-php

# 4. Quick test
sudo nano /var/www/html/test.php
```

```php
<?php phpinfo(); ?>
```

Visit `http://localhost/test.php`

```bash
# 5. Install MySQL
sudo apt install mysql-server
sudo mysql_secure_installation   # lock it down, say yes to everything

# 6. Install phpMyAdmin
sudo apt install phpmyadmin
# Select: apache2 (spacebar to check, then Enter)
# Configure with dbconfig-common: Yes

sudo phpenmod mbstring
sudo systemctl restart apache2
```

**phpMyAdmin:** `http://localhost/phpmyadmin`

**Give yourself write access to the web folder:**

```bash
sudo chown -R $USER:$USER /var/www/html
```

Now you can create PHP files without `sudo` every time.

## Method 3: Docker 

Docker keeps everything isolated in containers. Your main OS stays clean. The whole environment is defined in one file.

**Create project:**

```bash
mkdir my-php-project
cd my-php-project
mkdir src
```

**Create `docker-compose.yml`:**

```yaml
version: '3.8'

services:
  web:
    image: php:8.2-apache
    container_name: php_web
    ports:
      - "8080:80"
    volumes:
      - ./src:/var/www/html
    depends_on:
      - db

  db:
    image: mysql:8.0
    container_name: php_db
    environment:
      MYSQL_ROOT_PASSWORD: secret_root_password
      MYSQL_DATABASE: my_database
      MYSQL_USER: my_user
      MYSQL_PASSWORD: my_password
    ports:
      - "3306:3306"
    volumes:
      - db_data:/var/lib/mysql

  phpmyadmin:
    image: phpmyadmin/phpmyadmin
    container_name: php_pma
    environment:
      PMA_HOST: db
      PMA_PORT: 3306
    ports:
      - "8081:80"
    depends_on:
      - db

volumes:
  db_data:
```

**Create `src/index.php`:**

```php
<?php
echo "Hello from Docker!";
phpinfo();
?>
```

**Run everything:**

```bash
docker-compose up
```

|What|Where|
|---|---|
|PHP app|`http://localhost:8080`|
|phpMyAdmin|`http://localhost:8081`|
|DB username|`my_user`|
|DB password|`my_password`|

**Stop:** `Ctrl+C` or `docker-compose down`  
**Restart:** `docker-compose up`  
Database data is safe in the `db_data` volume between sessions.
