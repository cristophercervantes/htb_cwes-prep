**What is a function?**

A function is a block of code that you write once and run whenever you want by calling its name.

**How to define one:**

```php
function greetLearner() {
  echo "Hello, Learner!\n";
  echo "I hope you're enjoying PHP!\n";
  echo "Love, coding";
}
```

**Parts:**

|Part|What it does|
|:--|:--|
|`function`|Keyword that starts the definition|
|`greetLearner`|The name you use to call it later|
|`{ }`|Holds the code that runs|

**Naming rules:**

- Must start with a letter or `_`
    
- Can use letters, numbers, `_` after that
    
- Use **camelCase** for functions (`greetLearner`)
    
- Use **snake_case** for variables (`$user_name`) — just to tell them apart
    

**Why use functions?** So you don't rewrite the same code over and over. Define it once, call `greetLearner()` whenever you need it.

**Invoking (Calling) a Function**

Defining a function just saves it. It doesn't run anything.

To actually run it, you **invoke** it by writing its name followed by `()`:

```php
function greetLearner() {
  echo "Hello, Learner!\n";
  echo "I hope you're enjoying PHP!\n";
  echo "Love, php";
}

greetLearner();
```

**Output:**

```plain
Hello, Learner!
I hope you're enjoying PHP!
Love, php
```

`()` tells the computer "go run the code inside that function."

**Convention:** When talking about functions outside code (or in comments), always include `()` — like `greetLearner()`. That's how you know you're talking about a function, not a variable.

**Return Statements**

`return` sends a value back from a function so you can use it outside.

```php
function countdown() {
  echo "4, 3, 2, 1, ";
  return "blastoff!";
}

$return_value = countdown(); // Prints: 4, 3, 2, 1,
echo $return_value; // Prints: blastoff!
```

**What's happening:**

|Line|What happens|
|:--|:--|
|`echo "4, 3, 2, 1, ";`|Prints directly|
|`return "blastoff!";`|Sends the value back — doesn't print|
|`$return_value = countdown();`|Calls the function, captures the returned value|
|`echo $return_value;`|Prints the returned value|

`echo` inside the function prints immediately. `return` just hands the value back — you decide what to do with it.

**Why use return?** So functions can process data and give you the result, instead of just printing stuff you can't reuse.

**Return Stops the Function**

`return` doesn't just send a value back — it **immediately ends the function**. Anything after it won't run.

**Example 1 — return at the end:**

```php
function announceRunning() {
  echo "The first function is running!\n";
  return "This is the return value of the first function.";
}

$first_result = announceRunning();
echo $first_result;
```

**Output:**

```plain
The first function is running!
This is the return value of the first function.
```

Everything runs because `return` is last.

**Example 2 — return before echo:**

```php
function announceRunning2() {
  return "This is the return value of the second function.";
  echo "P.S., I love you"; // This never runs!
}

$second_result = announceRunning2();
echo $second_result;
```

**Output:**
```plain
This is the return value of the second function.
```

`"P.S., I love you"` never prints because `return` exits the function immediately.

Put `return` last, or anything after it is dead code.

**Return Values**

A returned value is just a value. You can use it anywhere you'd use a normal value.

```php
function returnFive() {
  return 5;
}

echo returnFive(); // Prints: 5
```

**How the computer sees it:**

```php
echo returnFive() + 3; // Prints: 8
```

The computer:

1. Runs `returnFive()` → gets `5`
    
2. Evaluates `5 + 3` → `8`
    
3. Prints `8`
    

Same as:

```php
$num = 5;
echo $num + 3; // Prints: 8
```

 Think of a function call as the value it returns. `returnFive()` _is_ `5` once it runs.

**Returning NULL**

If a function has no `return`, it returns `NULL` — a special value meaning "nothing."

```php
function returnNothing() {
  echo "I'm running! I'm running!\n";
}

$result = returnNothing(); // Prints: I'm running! I'm running!
echo $result; // Nothing is printed
```

**What's happening:**

|Step|Result|
|:--|:--|
|Call `returnNothing()`|Prints `"I'm running! I'm running!"`|
|No `return` in function|`$result` gets `NULL`|
|`echo $result;`|Nothing — `NULL` is empty|

No `return` = `NULL`. The function can still do stuff (like `echo`), but it gives back nothing.

**Parameters**

Parameters let you pass data into a function so it can do different things each time.

```php
function sayCustomHello($name) {
  echo "Hello, $name!";
}

sayCustomHello("Aisle Nevertell");   // Prints: Hello, Aisle Nevertell!
sayCustomHello("PHP learner"); // Prints: Hello, Codecademy learner!
```

**The parts:**

Table

|Term|What it is|
|:--|:--|
|Parameter|`$name` — the placeholder variable in the function definition|
|Argument|`"Aisle Nevertell"` — the actual value you pass in when calling|

**How it works:**

1. You define the function with `$name` as a parameter
    
2. When you call `sayCustomHello("Aisle Nevertell")`, PHP puts `"Aisle Nevertell"` into `$name`
    
3. The function runs with that value
    

One function, different outputs — just change the argument.

**Multiple Parameters**

You can pass more than one value into a function. Just separate them with commas.
```php
function divide($num_one, $num_two) {
  return $num_one / $num_two;
}

echo divide(12, 3);  // Prints: 4
echo divide(3, 12);  // Prints: 0.25
```

**Order matters:**

- First argument → first parameter (`$num_one`)
    
- Second argument → second parameter (`$num_two`)
    

**Important:** Pass the right number of arguments or you'll get an error.

```php
function expectTwo($first, $second) {
  return "whatever";
}

echo expectTwo("test"); // Error — missing second argument
```

Match the number and order of parameters when calling the function.

**Default Parameters**

You can set a fallback value for a parameter. If no argument is passed, it uses the default.

```php
function greetFriend($name = "old chum") {
  echo "Hello, $name!";
}

greetFriend("Marvin"); // Prints: Hello, Marvin!
greetFriend();         // Prints: Hello, old chum!
```

**Without a default, this breaks:**

```php
function sayCustomHello($name) {
  echo "Hello, $name!";
}

sayCustomHello(); // Error — missing argument
```

**With a default, it's safe:**

```php
function greetFriend($name = "old chum") {
  echo "Hello, $name!";
}

greetFriend(); // Uses "old chum", no error
```

`"value"` in the parameter sets the default. Pass an argument → it overrides. Don't pass one → default kicks in.

**Pass By Reference**

By default, PHP passes a **copy** of the value. Changes inside the function don't touch the original variable.
```php
function addX($param) {
  $param = $param . "X";
  echo $param;
}

$word = "Hello";
addX($word);    // Prints: HelloX
echo $word;     // Prints: Hello  ← unchanged
```

**To change the original, use `&`** — this makes the parameter point to the same memory as the argument.

```php
function addXPermanently(&$param) {
  $param = $param . "X";
  echo $param;
}

$word = "Hello";
addXPermanently($word); // Prints: HelloX
echo $word;             // Prints: HelloX  ← changed!
```

**The difference:**

|Without `&`|With `&`|
|:--|:--|
|Copies the value|References the same variable|
|Original stays the same|Original gets modified|

Add `&` before the parameter if you want the function to permanently change the variable you passed in.

**Variable Scope**

Variables inside a function live only inside that function. You can't access them from outside.

```php
function calculateDaysLeft($feed_quantity, $number, $rate) {
  $result = $feed_quantity / ($number * $rate);
  return $result;
}

echo calculateDaysLeft(300, 2, 30); // Works
echo $result; // Error — $result doesn't exist here
```

**Local scope:** `$result` is trapped inside the function.

**Global scope:** Use `global` to pull in an outside variable.

```php
$feed_quantity = 300;

function calculateDaysLeft($number, $rate) {
  global $feed_quantity;  // grabs the outside variable
  $result = $feed_quantity / ($number * $rate);
  return $result;
}

echo calculateDaysLeft(2, 120);
```

**Trade-off:** `global` saves you from passing arguments, but it's harder to tell what the function needs just by looking at it.

**Functions can call other functions:**
```php
function first() {
  echo "This works!\n";
}

function second() {
  first();  // calling another function
}

second(); // Prints: This works!
```

 Keep variables local when possible. Use `global` sparingly — it makes code harder to follow.
