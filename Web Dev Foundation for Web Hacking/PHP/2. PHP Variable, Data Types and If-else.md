# Variables & Data Types 

## Variables 

Think of a variable as a box with a label on it. You put something inside, stick a label on it, and come back to it later when you need it.

In PHP, every variable starts with a `$` dollar sign:

```php
$name = "Kiyotaka";
$age = 21;
$isAdmin = true;
```

 Every statement in PHP **must end with a semicolon** `;` — skip it and PHP throws an error.

## Data types 

|Type|Description|Example|
|---|---|---|
|String|Words / text|`"hello"`|
|Integer|Whole number|`10`|
|Float|Decimal number|`1.25`|
|Boolean|True or false|`true` / `false`|
|Array|A list of things|`array(1, 2, 3)`|

## Arrays 

```php
$fruits = array("apple", "banana", "mango");
// $fruits[0] = "apple"
// $fruits[1] = "banana"
// $fruits[2] = "mango"
```

Or with the short syntax:

```php
$fruits = ["Apple", "Banana", "Cherry"];
echo $fruits[0]; // Apple
```
Array indexing starts at **0**, not 1. First item is always index `0`.

## Named arrays (like a dictionary / key-value pairs)

```php
$person = array("name" => "John", "age" => 25);
// $person['name'] = "John"
// $person['age']  = 25
```

## Comments 

```php
// This is a single-line comment

/*
This is a multi-line comment.
It can span as many lines as you need.
Useful for documentation or disabling a block of code.
*/
```

## Basic PHP code structure

PHP lives inside special tags. The server processes everything between them:

```php
<?php
// your PHP code here
?>
```

A minimal "Hello World":

```php
<?php
echo "Hello World";
?>
```

## Mixing PHP with HTML

PHP files can contain both PHP logic and regular HTML. The server only processes what's inside the `<?php ?>` tags — everything else is passed straight to the browser unchanged.

```php
<!DOCTYPE html>
<html>
<body>

    <h1>Welcome</h1>

    <?php
    $name = "Alex";
    echo "<p>Hello, $name!</p>";
    ?>

    <p>This is plain HTML. The server ignores this.</p>

</body>
</html>
```

The browser receives clean HTML — it never sees the PHP tags.

# Output & User Input 

## Printing output 

### `echo` 

```php
echo "Hello World";
echo "My name is ", $name;  // can take multiple things
```

### `print` 

```php
print "Hello World";
```

### Putting variables inside text

```php
$name = "John";
print "Hello $name";   // Hello John
```

For array values inside strings, wrap in `{}`:

```php
$fruit = array("Apple", "Banana");
print "First fruit: {$fruit[0]}";
```

### `print_r()` 

```php
$fruits = array("apple", "banana");
print_r($fruits);
// Array ( [0] => apple [1] => banana )
```

Useful when you want to see the entire contents of an array.

### `printf()` 

```php
$i = 5;
$f = 3.14;
printf("Number: %d, Decimal: %2.2f", $i, $f);
// Number: 5, Decimal: 3.14
```

`%d` = integer placeholder, `%f` = float placeholder.

## Getting input from the user 

Whether through a URL or a form — PHP catches it automatically in special arrays called **superglobals**.

### `$_GET` — data from the URL

If someone visits:

```
http://localhost/hello.php?name=kiyotaka&age=21
```

PHP automatically creates:

```php
$_GET['name']  // = "kiyotaka"
$_GET['age']   // = "21"
```

The `?` starts the data. Each piece is `key=value`. Multiple pieces separated by `&`.

### `$_POST` — data from a form (invisible in URL)

When a form is submitted, data goes into:

```php
$_POST['username']   // whatever they typed
$_POST['password']   // whatever they typed
```

### `$_REQUEST` — catches both GET and POST

```php
$_REQUEST['name']   // works whether it came from URL or a form
```

### `$_COOKIE` — small data stored in the user's browser

Developers use cookies to remember things about a user between visits.

### `$_ENV` — server environment variables

```php
foreach(array_keys($_ENV) as $key) {
    print "$key = {$_ENV[$key]}\n";
}
```

`$_ENV` holds things like server usernames, system paths, and config info. Extremely useful for **recon** when you get code execution.

# Conditionals & Loops 

## Conditionals 

A conditional says: **if this is true, do this. Otherwise, do that.**

```php
$age = 20;

if($age >= 18) {
    print "You are an adult";
} else {
    print "You are a minor";
}
```

### Checking if user input exists

```php
if($_GET['name']) {
    print "Name was provided: " . $_GET['name'];
}
```

The `.` joins two strings together — this is called **concatenation**.

### AND / OR conditions

```php
// AND — both must be true
if($age > 18 && $name == "John") {
    // runs only if age > 18 AND name is John
}

// OR — at least one must be true
if($age > 18 || $name == "John") {
    // runs if either condition is true
}
```

### Checking if a function exists before calling it

```php
if(function_exists('php_uname')) {
    print php_uname();   // prints OS info: "Linux server 5.4.0 ..."
}
```

Some servers **disable dangerous functions**. Checking first prevents errors — and also tells you what the server has available.

## Loops 

### `foreach` — go through every item in a list

```php
$fruits = array("apple", "banana", "mango");

foreach($fruits as $fruit) {
    print $fruit . "\n";
}
// apple
// banana
// mango
```

With a named array (key => value):

```php
foreach(array_keys($_ENV) as $key) {
    print "$key = {$_ENV[$key]}\n";
}
```

This dumps every server environment variable — useful for recon.

### `for` loop — repeat a fixed number of times

```php
for($i = 0; $i < 5; $i++) {
    print $i . "\n";
}
// 0, 1, 2, 3, 4
```

Breaking it down:

- `$i = 0` — start here
- `$i < 5` — keep going while this is true
- `$i++` — increment by 1 each time (`$i++` is shorthand for `$i = $i + 1`)

### `while` loop — repeat until a condition is false

```php
$i = 0;
while($i < 3) {
    print $i . "\n";
    $i++;
}
// 0, 1, 2
```

### `continue` — skip one iteration

```php
for($i = 0; $i < 5; $i++) {
    if($i == 3) { continue; }   // skip when i = 3
    print $i . "\n";
}
// 0, 1, 2, 4   (3 is skipped)
```
